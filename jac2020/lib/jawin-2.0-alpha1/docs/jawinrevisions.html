Jawin Revision History

  • 23/03/2005 Version 2.0, alpha 1
    • Wrapped all JNI-calls in try-catch blocks catching all native exceptions and converting them to Java Exceptions, using Konstantin Boukreevs C++ Exception Handling classes (Morten Andersen).
    • Updated the thread-section in the architecture document so it is aligned with the actual implementation (Morten Andersen).
    • Major code restructering and cleanup together with additional documentation - see the jawinuserguide_*.html files in the docs-folder (Morten Andersen).
    • Added documentation about how to do in-process interop with .NET. - see docs/jawinuserguide_net.html (Morten Andersen).
    • Fixed problem with initialization of Interface constants, that kept Jawin from working with IBM's JRE implementations, used in eg. WebSphere. See this thread for further explanations (Steve Doyle and Morten Andersen).
    • Fixed native exception when passing SafeArrays containing BSTR's to IDispatch methods (Tobias Rohatsch and Morten Andersen).
    • Fixed native exception when passing SafeArrays containing SafeArrays (that is java Object[][]) to IDispatch methods (Tobias Rohatsch and Morten Andersen).
    • Fixed leak problem when IDispatch methods returns SafeArrays (Morten Andersen).
    • Removed the last of the deprecated com.develop-source - both C++ and Java (Morten Andersen).
    • Support for 1 dimensional SafeArrays in Dispatch calls (Morten Andersen).
    • Support for out parameters in Dispatch calls (Morten Andersen).
    • Finished(?) support for out parameters in Dispatch calls - see the class org.jawin.Variant.ByrefHolder (Morten Andersen).
    • Cleaned some of the marshalling code (Morten Andersen).
    • Added Win32 version info to the DLL-files (Morten Andersen).
    • Sealed the jawin.jar and jawinBrowser.jar repositories (Morten Andersen).
    • Added support for VT_DATE to the Jawin Type Browser (Morten Andersen).
    • Moved "non-core"-Jawin classes out of the org.jawin.win32 -package and into a donated stubs library (this is primarely the handcoded WIN32-API DLL-stubs used by the demos) (Morten Andersen).
    • Updated the unit tests to test the org.jawin code instead of the com.develop.jawin code, as well as updated the documentation section about it. Finally some indicative performance tests was added (Morten Andersen).
  • 26/05/2004 Version 1.0.19
    • Added a concrete implementation of IUnknown (org.jawin.UnknownPtr), necessary when IDispatch methods returns VT_UNKNOWN objects (Ken Larson).
    • Updated the Ant build process for the native code to work directly with the Visual Studio project files. Done by replacing the nmake calls with calls to msdev.exe (Robert Hastings).
    • Deprecated the com.develop.jawin source and moved to org.jawin (Roger I Martin and Paul Marquis).
    • New tool for stub generation: Jawin Type Browser (Josh Passenger and Roger I Martin).
    • Refactored documentation to document the org.jawin branch (Morten Andersen).
    • Refactored build to match the org.jawin branch (Morten Andersen).
  • 11/19/2002 Version 1.0.18
    • Stuart Halloway checked Jawin into sourceforge.
    • Updated build process to exec an nmake for native code. Finally, the whole build is automated!
  • 11/19/2002 Version 1.0.17
    • David Handy and Stuart Halloway added unit test for VARIANT SAFEARRAY marshalling and fixes bugs in same.
    • Also changed JUnit test compile to include debug information by default.
    • Fixed build script so test objects get registered before tests run.
  • 11/06/2002 Version 1.0.16 Stuart Halloway added documentation for writing unit tests and ported Jawin unit tests to use JUnit
  • 02/25/2001 Version 1.0.15 Stuart Halloway documented the need for XSLT processor to run Jawin build
  • 02/16/2001 Version 1.0.14 Stuart Halloway added support for [out] SAFEARRAYs (for the more common data types, anyway)
  • 02/14/2001 Version 1.0.13 Stuart Halloway added support for IEnumVariant. Along the way, also
    • added convenience accessor getCount() to NakedByteStream
    • added marshal instructions 'r' and 'L' with negative args
  • 01/29/2001 Version 1.0.12 Stuart Halloway fixed examples that did not correctly catch COMExceptions
  • 01/19/2001 Version 1.0.11 Stuart Halloway prototype documentation for instruction strings
  • 01/19/2001 Version 1.0.10 Stuart Halloway corrected doc error (links to outdated jawinoverview.html page)
  • 12/17/2001 Version 1.0.9 Stuart Halloway added support for VARIANTs of type VT_DATE
  • 12/16/2001 Version 1.0.8 Andrew Wason added unit test for indexed properties, marshalling for variants containing reals and doubles
  • 12/06/2001 Version 1.0.7 Andrew Wason added support for indexed properties. See the ExcelProp Ant task and changes to the DispatchPtr class
  • 12/04/2001 Version 1.0.6 inserts Registry.RegEnumKey provided by Curt Hagenlocher
  • 12/03/2001 Version 1.0.5 fixes bug in Variant.marshalInArray, code had been looking for arrays of wrappers instead of arrays of primitives. Added unit tests for array of int, short, boolean. Thanks to Miguelângelo Faria Rocha for pointing out the problem.
  • 11/28/2001 Modified Transform::badMarshalString to throw an exception instead of logging to a file. Fixed bad trailing characters on marshal strings in various Ole32 methods.
  • 11/28/2001 Modified COMPtr behavior to fix reference-counting problem pointed out by Andrew Wason on the mailing list.
  • 11/24/2001 Added support for creating classes by ProgID without requiring use of the new moniker, i.e.
    DispatchPtr app = new DispatchPtr("PowerPoint.Application");
    //NOT: DispatchPtr app = new DispatchPtr("new:PowerPoint.Application");
  • 2/17/2001 Another fix from Gary. Replaced the implementation of initFuncPtr with a version that was Win95/98 compatible. Also, Stu added some new registry APIs and shared stubs to support them.
  • 2/16/2001 several fixes suggested by Gary Gregory and Ivan Filippenko. Switched to Ant build system, added version information to build process, separated common code into DM.jar, and removed tests (such as TestOle32.smokeTest) that require special software.
  • 9/21/2000 extended security example to show user SID.
